# Contributing to Email Inbox Manager

Thanks for your interest in contributing! This is an open-source AI skill, and we welcome 
contributions of all kinds.

## Ways to Contribute

### 🐛 Bug Reports & Feature Requests

Open an issue describing:
- What you expected to happen
- What actually happened
- Your environment (Cowork, Claude Code, or claude.ai)
- Any relevant parts of your governance map (redact personal info!)

### 📋 Governance Map Templates

One of the most valuable contributions is **persona-specific governance map templates**. 
If you've set up the skill for your workflow and think others in a similar role would 
benefit, consider sharing a sanitized version of your governance map.

Templates we'd love to see:
- Solopreneur / one-person business
- VP or Director with an EA/assistant
- Founder with a VA team
- Sales professional
- Consultant managing multiple clients

To contribute a template:
1. Copy your `email-governance-map.yaml`
2. Remove all personal data (email addresses, sender names, etc.)
3. Replace with realistic but fictional examples
4. Add it to `templates/` with a descriptive name like `template-solopreneur.yaml`
5. Submit a PR

### 🔧 Skill Improvements

If you've found that the SKILL.md instructions could be clearer, or you've discovered 
a better prompting pattern for a specific mode, PRs are welcome. Please describe what 
you changed and why in your PR description.

### 📖 Documentation

Improvements to the README, additional examples, or how-to guides are always appreciated.

## How to Submit a Pull Request

1. Fork the repo
2. Create a branch (`git checkout -b my-improvement`)
3. Make your changes
4. Test with your own inbox (if modifying the skill)
5. Commit with a clear message
6. Push and open a PR

## Code of Conduct

Be kind, be constructive, and remember that everyone organizes their email differently — 
that's literally the point of this project.

## Questions?

Open a Discussion (not an Issue) for general questions about using the skill.
